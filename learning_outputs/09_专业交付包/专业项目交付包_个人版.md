# AI Solution Consultant 专业项目交付包：个人版

项目名称：客服工单分类与回复建议 Agent

客户 / 部门：模拟客服部门

负责人：个人版

日期：2026-06-12

---

## 1. 项目背景

客服团队每天需要处理大量客户问题，包括付款异常、物流咨询、退款申请、账号问题和投诉。当前主要依赖人工阅读、分类、查询资料和回复，存在处理效率低、分类标准不一致、高风险工单漏判等问题。

本项目设计一个 Agentic AI 原型，辅助客服完成工单分类、优先级判断、知识库检索、回复草稿生成和高风险转人工。

## 4. As-Is 流程

```text
客户提交工单
  -> 客服系统记录问题
  -> 一线客服人工阅读工单
  -> 判断类别和优先级
  -> 查询知识库或历史工单
  -> 手写回复内容
  -> 判断是否需要升级二线
  -> 发送回复或转交二线
  -> 关闭工单并记录结果
```

---

## 5. To-Be 流程

```text
客户提交工单
  -> Agent 读取工单内容
  -> Agent 分类、判断优先级和风险等级
  -> Agent 检索知识库和历史工单
  -> Agent 生成回复草稿
  -> 高风险工单转人工确认
  -> 客服确认或修改回复
  -> 系统记录 Agent 输出、工具调用、审批和最终结果
```

## 6. Agent PRD

### 6.1 目标用户

- 一线客服
- 二线客服
- 客服主管

### 6.2 业务目标

- 降低工单初步处理时间。
- 提高分类一致性。
- 提高高风险工单识别率。
- 降低客服查询知识库的时间。

### 6.3 输入

- 工单 ID
- 客户 ID
- 工单文本
- 渠道
- 客服用户 ID

### 6.4 输出

- 工单类别
- 优先级
- 是否需要人工确认
- 回复草稿
- 风险原因
- 引用来源
- 工具调用记录

### 6.5 工作流

```text
输入工单
  -> 分类和优先级判断
  -> 查询客户信息
  -> 检索知识库
  -> 生成回复草稿
  -> 判断是否需要人工确认
  -> 记录日志
```

### 6.6 边界

Agent 做：

- 分类
- 优先级判断
- 检索知识库
- 生成回复草稿
- 建议是否转人工
- 记录处理理由

Agent 不做：

- 不自动退款
- 不自动发送正式回复
- 不修改客户资料
- 不关闭高风险工单
- 不绕过权限查询数据

## 7. 系统架构

```text
客服系统 / Web 入口
  -> 身份认证和权限校验
  -> Agent 编排服务
  -> LLM 分类和回复生成
  -> RAG 知识库检索
  -> 工具调用层
      -> 查询客户信息
      -> 查询历史工单
      -> 创建升级工单
  -> 人工审批
  -> 日志、审计、评估
```

## 8. RAG / 工具设计

### 8.1 RAG 设计

| 文档 | 权限 | 更新频率 | 负责人 |
|---|---|---|---|
| 客服常见问题 | 客服内部 | 每月 | 客服主管 |

### 8.2 工具设计

| 工具 | 用途 | 输入 | 输出 | 风险 | 审批 |
|---|---|---|---|---|---|
| get_customer_profile | 查询客户信息 | customer_id, ticket_id | 客户等级、风险标签 | 中 | 否 |
| search_support_knowledge_base | 检索知识库 | query, role | 文档片段 | 低到中 | 否 |
| search_similar_tickets | 查询历史工单 | ticket_text | 相似工单 | 中 | 否 |
| draft_customer_reply | 生成回复草稿 | ticket_text, category | 回复草稿 | 中到高 | 是 |
| create_escalation_ticket | 创建升级工单 | ticket_id, reason | 升级 ID | 中 | 条件审批 |

## 9. 权限和风险治理

| 风险 | 后果 | 控制措施 | 负责人 |
|---|---|---|---|
| 付款问题误判 | 客户投诉或处理延迟 | 付款类强制转人工 | 客服主管 |
| 回复草稿过度承诺 | 客户误解或法律风险 | 发送前人工确认 | 一线客服 |
| 越权查询客户数据 | 数据泄露 | 工具调用前权限校验 | IT |
| 敏感信息进入模型 | 合规风险 | 脱敏和字段白名单 | 安全负责人 |
| Prompt Injection | 工具误调用或泄密 | 注入测试和系统指令隔离 | IT / 安全 |
