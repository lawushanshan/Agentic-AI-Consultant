---

## 2. Prompt 写法原则

每个 Prompt 尽量包含：

- 角色：让模型知道自己扮演什么业务角色
- 背景：说明业务场景
- 任务：明确要完成什么
- 输入：提供待处理内容
- 输出格式：优先使用 JSON、表格或固定字段
- 限制条件：说明不能编造、不能越权、何时转人工
- 示例：必要时给一个输出样例

### 案例 1：客服工单分类

#### 任务类型

分类 / 风险判断 / 结构化输出

#### 原始 Prompt

```text
请判断下面客户问题的类别和优先级：
客户说：“我已经付款了，但是订单还是显示未支付。”
```

#### AI 输出结果

```text
待填写
```

#### 存在问题

- 输出格式不稳定。
- 没有说明是否需要人工介入。
- 没有给出业务判断依据。

#### 改进后的 Prompt

```text
你是一名客服主管。
背景是：我们需要把客户工单分配给不同客服小组，并识别高风险问题。
你的任务是：判断下面客户问题的类别、优先级、是否需要人工处理，并给出理由。

客户问题：
“我已经付款了，但是订单还是显示未支付。”

请严格按照 JSON 输出：
{
  "category": "payment_issue | logistics_issue | refund_issue | account_issue | other",
  "priority": "low | medium | high",
  "need_human": true,
  "reason": "判断理由"
}

限制条件：
- 涉及付款、退款、投诉的问题，优先级至少为 medium。
- 涉及付款异常的问题必须 need_human = true。
- 不要编造订单状态，只能根据输入判断。
```

```text

```

### 案例 2：企业制度问答拒答

#### 任务类型

RAG 问答 / 无答案拒答

#### 原始 Prompt

```text
请回答员工年假可以累计几年。
```

#### AI 输出结果

```text
待填写
```

#### 存在问题

- 没有限定必须基于制度文档回答。
- 没有要求引用来源。
- 可能根据常识编造。

#### 改进后的 Prompt

```text
你是企业制度问答助手。
你只能基于提供的制度片段回答问题。
如果制度片段中没有答案，请回答“根据当前资料无法确认”，不要编造。

制度片段：
【待粘贴文档片段】

用户问题：
员工年假可以累计几年？

请按照以下格式输出：
答案：
引用来源：
是否能确认：是/否
```

#### 改进后的输出结果

```text
待填写
```

### 案例 3：销售跟进邮件草稿

#### 任务类型

邮件生成 / 人工审批

#### 原始 Prompt

```text
帮我写一封销售跟进邮件。
```

#### AI 输出结果

```text
待填写
```

#### 存在问题

- 缺少客户背景。
- 缺少语气要求。
- 没有提醒人工确认。
- 容易生成过度承诺。

#### 改进后的 Prompt

```text
你是一名 B2B 销售助理。
背景是：销售需要跟进一个可能续约的企业客户。
你的任务是：根据客户信息生成一封跟进邮件草稿。

客户信息：
- 公司：待填写
- 最近一次采购：待填写
- 已知问题：待填写
- 跟进目标：预约一次沟通会议

请输出：
邮件主题：
邮件正文：
需要销售人工确认的内容：

限制条件：
- 不要承诺价格、折扣、交付日期。
- 不要替销售直接发送邮件。
- 邮件语气专业、简洁。
```

#### 改进后的输出结果

```text
待填写
```
