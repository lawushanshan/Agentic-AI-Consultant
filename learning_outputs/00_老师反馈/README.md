# 老师反馈使用说明

这个目录用于记录每周学习产出的点评、修改要求和二次验收结果。

它不是新的学习手册，而是学习闭环记录：

```text
提交本周产出
  -> 老师点评
  -> 学员修改
  -> 记录修改动作
  -> 二次验收
```

---

## 1. 每周怎么用

每周至少保留 1 份反馈文件，例如：

```text
第04周_老师反馈.md
第05周_老师反馈.md
第06周_老师反馈.md
```

每份反馈文件都应该回答 4 个问题：

1. 本周提交了什么？
2. 老师认为哪里合格、哪里不合格？
3. 必须修改哪 3-5 个点？
4. 修改后是否通过二次验收？

---

## 2. 老师点评原则

点评时只看可检查证据，不按“看起来很努力”评分。

优先检查：

- 是否有具体业务场景
- 是否有输入、输出、流程、数据或接口样例
- 是否记录失败案例
- 是否有测试结果
- 是否体现权限、审批、日志和风险控制
- 是否能作为作品集材料复用

---

## 3. 学员修改要求

每次收到反馈后，不要只在反馈文件里写“已修改”。

必须同时修改对应产出文件，并在反馈文件里记录：

| 修改项 | 对应文件 | 修改说明 | 是否完成 |
|---|---|---|---|
| 示例 | `02_Prompt练习/Prompt练习集_个人版.md` | 补充 5 个 Prompt 案例 | 是 |

---

## 4. 通过标准

一周学习通过，至少满足：

- 本周核心产出已更新
- 至少 1 个失败案例或改进记录
- 涉及项目时至少 5 条测试数据
- 老师反馈中的关键修改项已完成
- 二次验收结果为“通过”或“有条件通过”
