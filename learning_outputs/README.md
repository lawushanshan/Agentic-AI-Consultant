# learning_outputs 使用说明

这个目录用于存放学习过程中的实际产出模板。学习者不要只阅读教程，要按阶段填写这些模板。

---

## 1. 模板对应关系

| 目录 | 模板 | 对应学习阶段 | 用途 |
|---|---|---|---|
| `01_职业理解` | `职业理解一页纸_模板.md` | 零基础第 1 周 | 建立职业认知 |
| `02_Prompt练习` | `Prompt练习集_模板.md` | 零基础第 3-4 周 | 记录 Prompt 改进过程 |
| `03_业务流程` | `业务流程分析_模板.md` | 零基础第 5-6 周 | 拆解业务流程和 AI 介入点 |
| `04_RAG原型` | `RAG原型说明_模板.md` | 零基础第 7 周 | 记录知识库问答原型 |
| `05_工具调用` | `工具设计卡片_模板.md` | 零基础第 8-9 周 | 设计 Agent 可调用工具 |
| `06_风险控制` | `风险控制清单_模板.md` | 零基础第 10 周 | 识别权限、审批和禁止动作 |
| `07_综合项目` | `Agent综合项目说明_模板.md` | 零基础第 11 周 | 完成一个可讲解的小项目 |
| `08_作品集` | `入门作品集_模板.md` | 零基础第 12 周 | 汇总求职作品集 |
| `09_专业交付包` | `专业项目交付包_模板.md` | 专业交付训练 | 输出完整客户项目交付包 |
| `10_验收评分表` | `学习验收评分表.md` | 阶段验收 | 教学者或自评使用 |

---

## 2. 新人使用方法

新人按这个顺序填写：

```text
01_职业理解
  -> 02_Prompt练习
  -> 03_业务流程
  -> 04_RAG原型
  -> 05_工具调用
  -> 06_风险控制
  -> 07_综合项目
  -> 08_作品集
  -> 10_验收评分表
```

每周至少完成一个模板。不要等到最后一起补。

---

## 3. 专业人员使用方法

专业人员可以直接使用：

```text
09_专业交付包
  -> 10_验收评分表
```

如果发现自己在某些基础部分薄弱，再回到对应模板补练习。

---

## 4. 填写要求

合格的模板应该满足：

- 不是空泛描述，有具体业务场景
- 有输入、输出、流程或数据样例
- 有失败案例或风险说明
- 有明确下一步改进
- 能被别人看懂，不依赖口头解释

---

## 5. 建议保留多个版本

建议每个学习者复制模板后另存为自己的版本，例如：

```text
职业理解一页纸_张三_2026-06-11.md
客服工单Agent综合项目_张三_v1.md
客服工单Agent综合项目_张三_v2.md
```

保留版本可以看到自己的改进过程，也方便后续整理作品集。
