# learning_outputs 使用说明

这个目录用于存放学习过程中的实际产出模板。学习者不要只阅读教程，要按阶段填写这些模板。

当前目录已增加一套 `AI Solution Consultant 个人版` 产出文件，适合有 ASP.NET Core / 后端开发经验、正在向技术型 AI Solution Consultant 转型的人使用。

个人版总入口：

```text
AI_Solution_Consultant_个人版产出目录.md
```

## 2. 新人使用方法

新人按这个顺序填写：

```text
01_职业理解
  -> 02_Prompt练习
  -> 03_业务流程
  -> 04_RAG原型
  -> 05_工具调用
  -> 06_风险控制
  -> 07_综合项目
  -> 08_作品集
  -> 11_导师评审
  -> 10_验收评分表
```

每周至少完成一个模板。不要等到最后一起补。

---

## 3. 专业人员使用方法

专业人员可以直接使用：

```text
09_专业交付包
  -> 10_验收评分表
```

如果发现自己在某些基础部分薄弱，再回到对应模板补练习。

---

## 4. 填写要求

合格的模板应该满足：

- 不是空泛描述，有具体业务场景
- 有输入、输出、流程或数据样例
- 有失败案例或风险说明
- 有明确下一步改进
- 能被别人看懂，不依赖口头解释

完成关键产出后，建议调用 AI 学习导师评审一次，再更新验收评分表。不要只靠自我感觉判断是否合格。

---

## 5. 建议保留多个版本

建议每个学习者复制模板后另存为自己的版本，例如：

```text
职业理解一页纸_张三_2026-06-11.md
客服工单Agent综合项目_张三_v1.md
客服工单Agent综合项目_张三_v2.md
```

保留版本可以看到自己的改进过程，也方便后续整理作品集。
