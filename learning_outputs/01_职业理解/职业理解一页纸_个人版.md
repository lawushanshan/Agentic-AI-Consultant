# 职业理解一页纸

填写日期：2026-06-12

学习者：个人版

---

## 1. 我理解的 Agentic AI Consultant 是什么

Agentic AI Consultant 不是单纯写 Prompt，也不是只做 AI 演示，而是把企业真实业务问题、AI Agent 技术、系统集成、权限治理、测试评估和项目交付连接起来的人。

结合我的程序员背景，我更适合走技术型 AI Solution Consultant 路线：既能写代码做 PoC，也能和业务方沟通需求，分析流程痛点，设计 RAG、工具调用、审批、日志和上线方案，帮助企业把 AI 从 Demo 落到真实业务流程里。

我的目标不是从程序员完全转成纯咨询顾问，而是从 ASP.NET Core 后端程序员升级为能做大模型应用方案和 Agentic AI 项目交付的技术型顾问。

---

## 2. 这个岗位主要解决什么问题

- 问题 1：企业有很多 AI 想法，但不知道哪些场景真正适合落地，需要有人判断业务价值、数据可得性、风险和技术可行性。
- 问题 2：很多 AI Demo 只能聊天或演示，不能接入真实系统，需要有人把 Agent、RAG、工具调用和企业 API、数据库、权限体系连接起来。
- 问题 3：企业使用 AI 时存在误操作、越权、数据泄露、结果不可评估等风险，需要有人设计审批、日志、测试集、评估指标和上线治理方案。

---

## 3. 这个岗位每天可能做什么

- 项目前期：和业务方、产品、运营或客户沟通，理解业务流程，识别高频、低效、重复、需要判断的 AI 场景。
- 方案设计：画 As-Is / To-Be 流程，设计 Agent PRD、RAG 方案、工具调用、权限边界、审批节点和评估指标。
- PoC 阶段：使用 ASP.NET Core、Python 或其他工具快速开发原型，调用模型 API，接入模拟数据或真实接口，验证流程是否跑通。
- 上线阶段：对接身份认证、数据库、业务系统、日志、权限、部署、测试和回滚，确保 AI 应用可控上线。
- 持续运营：分析日志、失败案例、用户反馈、成本、延迟和任务成功率，持续优化 Prompt、知识库、工具和流程。

## 4. 需要哪些能力

| 能力 | 我当前水平 | 需要怎么补 |
|---|---|---|
| 业务理解 | 有一定基础。接触过客户、产品、运营或业务方，也做过企业业务系统。 | 进一步训练流程诊断、场景评分、ROI 估算和业务价值表达。 |
| Prompt | 有 AI 使用基础，但还需要系统训练。 | 完成至少 20 个业务 Prompt 案例，记录改进前后效果。 |
| RAG | 需要重点补齐。 | 做一个企业知识库问答 Agent，掌握文档切分、检索、引用、无答案拒答和权限设计。 |
| 工具调用 | 有后端 API 基础，具备优势。 | 把已有 API 能力升级为 Agent 工具设计能力，补齐工具 schema、风险等级、审批和日志。 |
| 安全治理 | 有日志、权限、审批、测试经验，基础较好。 | 学会把这些经验迁移到 AI Agent 场景，重点关注越权调用、敏感数据、Prompt Injection 和审计。 |
| 项目交付 | 有企业系统、部署和技术方案表达基础。 | 补齐 Agent PRD、架构图、测试集、评估报告、上线计划和专业交付包。 |

---

## 5. 我和这个岗位的差距

目前最大的 3 个差距：

1. AI 应用专项能力还不够系统：需要补齐 LLM API、RAG、Agent 工作流、工具调用和评估体系。
2. 顾问式交付表达还需要强化：不能只展示代码，还要能输出流程图、PRD、架构图、风险清单、测试报告和 ROI 说明。
3. 行业场景沉淀不足：需要选择 1-2 个方向深入，例如企业知识库、客服工单、销售 CRM、采购或财务流程。

---

## 6. 接下来 12 周学习目标

- 第 1 个目标：完成 Prompt 和结构化输出训练，做出一个可以调用模型 API 的小型业务分类服务。
- 第 2 个目标：完成企业知识库问答 Agent，包含文档检索、引用来源、无答案拒答、测试问题和错误案例复盘。
- 第 3 个目标：完成客服工单分类或销售续约分析 Agent，包含工具调用、人工确认、日志、测试集和风险控制说明。

## 8. 我的推荐转型路线

```text
ASP.NET Core 企业系统程序员
  -> LLM 应用开发者
  -> AI Solution Engineer
  -> 技术型 AI Solution Consultant
  -> Agentic AI Solution Architect / Forward Deployed AI Engineer
```

短期不要放弃工程优势。我的核心竞争力应该是：

> 用工程能力做出可运行 AI PoC，用业务理解找到真实场景，用交付方法把 Demo 变成可评估、可上线、可治理的 Agentic AI 系统。
