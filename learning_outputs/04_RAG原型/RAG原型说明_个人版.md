# RAG 原型说明：个人版

填写日期：2026-06-12

学习者：ASP.NET Core 程序员转 AI Solution Consultant

原型名称：企业知识库问答 Agent

---

## 1. 使用场景

本原型用于模拟企业内部知识库问答。用户可以询问制度、流程、产品说明或客服 FAQ，Agent 必须先检索资料，再基于资料回答，并返回引用来源。如果资料中没有答案，Agent 必须拒答，不能编造。

训练目标：

- 理解 RAG 流程
- 练习文档切分和检索
- 设计引用来源
- 设计无答案拒答
- 思考文档权限和版本管理

## 4. RAG 流程设计

```text
文档收集
  -> 文档清洗
  -> 文档切分
  -> 元数据标注
  -> embedding
  -> 存入向量索引
  -> 用户提问
  -> 检索相关片段
  -> 组装上下文
  -> 模型基于上下文回答
  -> 返回答案和引用来源
```

## 7. 权限和风险

- 所有人可访问：公开产品说明、普通 FAQ。
- 内部员工可访问：请假制度、报销流程、售后规则。
- 需要限制权限：客户数据、合同、工资、人事、财务敏感数据。
- 不应进入模型上下文：密钥、完整身份证号、银行卡号、未授权客户清单。
- 找不到答案时应该回答：根据当前资料无法确认，请联系对应负责人。
