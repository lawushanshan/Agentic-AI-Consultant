# AI Solution Consultant 24 周学习日程

起始日期：2026-06-15

适合节奏：每周 8-10 小时

目标：

> 用 24 周完成从 ASP.NET Core 程序员到技术型 AI Solution Consultant 候选人的第一轮训练，最终产出 2 个可展示项目、1 个入门作品集、1 个专业项目交付包。

---

## 1. 每周固定节奏

建议每周按下面节奏执行：

| 时间 | 内容 | 要求 |
|---|---|---|
| 周一 | 学习本周概念 | 阅读教程，明确本周目标 |
| 周二 | 做第 1 轮练习 | 写 Prompt、流程、代码或测试数据 |
| 周三 | 做第 2 轮练习 | 继续实现功能或完善文档 |
| 周四 | 测试和记录失败 | 记录错误案例、风险、改进点 |
| 周五 | 整理交付物 | 更新 `learning_outputs` 对应文件 |
| 周末 | 复盘和补缺 | 用 1-2 小时修正本周产出 |

每周最低交付要求：

- 至少更新 1 个 Markdown 文件
- 至少记录 1 个失败或改进案例
- 如果涉及项目，至少保留 5 条测试数据

## 3. 第 1-4 周：LLM 应用基础

### 第 1 周：Prompt 基础

日期：2026-06-15 至 2026-06-21

目标：

- 理解业务 Prompt 的基本结构
- 完成 10 个业务 Prompt 案例
- 记录改进前后效果

任务：

1. 阅读 `AI_Solution_Consultant_个人转型教程.md` 的阶段 1。
2. 新建 `learning_outputs/02_Prompt练习/Prompt练习集_个人版.md`。
3. 完成 10 个 Prompt，覆盖总结、分类、提取、改写、邮件生成、风险判断。
4. 至少选择 3 个 Prompt 做改进前后对比。

本周交付：

- `Prompt练习集_个人版.md`
- 10 个 Prompt 案例
- 3 个失败和改进记录

验收标准：

- 每个 Prompt 有角色、背景、任务、输入、输出格式和限制条件。
- 不是只贴 AI 输出，而是写清楚哪里不好、怎么改。

### 第 2 周：结构化输出

日期：2026-06-22 至 2026-06-28

目标：

- 学会让模型输出固定 JSON
- 设计业务系统可消费的字段

任务：

1. 设计 5 个结构化输出任务。
2. 每个任务定义字段、类型、枚举值和必填项。
3. 测试模型是否稳定输出 JSON。
4. 记录字段缺失、格式错误、内容不一致的案例。

推荐任务：

- 客服工单分类
- 报销申请风险判断
- 简历信息提取
- 会议纪要行动项
- 合同条款风险抽取

本周交付：

- 5 个 JSON 输出案例
- 字段设计说明
- 失败案例记录

验收标准：

- JSON 字段能被后端程序直接解析。
- 能说明哪些字段应该使用枚举值。

### 第 3 周：LLM API 调用

日期：2026-06-29 至 2026-07-05

目标：

- 写一个最小可运行的模型 API 调用程序
- 把 Prompt 训练和代码结合起来

任务：

1. 用 Python 或 ASP.NET Core 写一个命令行 AI 小助手。
2. 输入客服问题，调用模型 API。
3. 输出类别、优先级、是否转人工、理由。
4. 保存输入、输出、错误日志。
5. 准备 10 条测试输入。

本周交付：

- 命令行 AI 小助手
- 10 条测试输入
- 运行说明 README

验收标准：

- 能稳定调用模型 API。
- 输出是结构化 JSON。
- 出错时有错误提示和日志。

### 第 4 周：AI Web API

日期：2026-07-06 至 2026-07-12

目标：

- 把命令行程序封装成 Web API
- 形成第一个小型 AI 后端服务

任务：

1. 创建一个 `/classify-ticket` 接口。
2. 输入 JSON，输出 JSON。
3. 增加基础参数校验。
4. 增加日志记录。
5. 用 10 条测试数据验证。

本周交付：

- AI 分类 API Demo
- 接口说明
- 10 条 API 测试记录

验收标准：

- 接口可以被 Postman、curl 或前端调用。
- 有输入校验、异常处理和日志。
